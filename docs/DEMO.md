# LwChat 演示脚本

## 1. 启动后端

打开终端，进入后端目录：

```bash
cd backend
python -m pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 8000
```

确认后端运行地址：

```txt
http://localhost:8000
```

## 2. 启动前端

打开第二个终端，进入前端目录：

```bash
cd web
npm install
npm run dev
```

确认前端运行地址：

```txt
http://localhost:5173
```

## 3. 注册两个测试用户

1. 在普通浏览器打开 `http://localhost:5173`。
2. 点击注册入口。
3. 注册用户：

```txt
用户名：user1
密码：123456
昵称：用户1
```

4. 打开无痕窗口或另一个浏览器。
5. 注册用户：

```txt
用户名：user2
密码：123456
昵称：用户2
```

## 4. 两个浏览器窗口分别登录

- 普通浏览器登录 `user1`。
- 无痕窗口登录 `user2`。
- 说明页面刷新后会通过 `localStorage` 尽量保持当前用户状态。

## 5. 演示私聊文本消息

1. 在 `user1` 页面左侧选择 `user2`。
2. 在输入框输入一条文本消息，例如“你好，这是私聊消息”。
3. 按 Enter 或点击发送。
4. 切换到 `user2` 页面，确认消息实时出现。
5. 说明消息通过 WebSocket 实时推送，并保存到 SQLite。

## 6. 演示私聊图片消息

1. 在 `user2` 页面选择 `user1` 会话。
2. 点击“图片”按钮，选择本地图片。
3. 等待上传状态结束。
4. 切换到 `user1` 页面，确认图片实时显示。
5. 说明图片先通过 HTTP 上传，再通过 WebSocket 发送图片消息。

## 7. 演示创建群聊

1. 在 `user1` 页面左侧输入群聊名称，例如“课程演示群”。
2. 点击创建。
3. 页面自动进入群聊。
4. 右侧信息栏显示群公告、群成员和群主信息。

## 8. 演示群聊消息

1. 在 `user2` 页面点击左侧的“课程演示群”。
2. 系统会加入群聊并进入聊天窗口。
3. `user1` 和 `user2` 分别发送群消息。
4. 确认两个窗口都能实时看到群消息。

## 9. 演示刷新后加载历史消息

1. 刷新 `user1` 页面。
2. 重新选择 `user2` 私聊或群聊。
3. 确认之前发送的文本消息和图片消息仍然存在。
4. 说明系统通过 SQLite 保存历史消息，刷新后通过 HTTP 接口加载。

## 10. 总结系统功能

演示结束时可以总结：

- LwChat 支持注册、登录和退出。
- 支持一对一私聊和群聊。
- 支持文本消息和图片消息。
- 支持 WebSocket 实时通信。
- 支持历史消息保存和刷新后加载。
- 支持群公告、群成员和群主基础管理。
- 项目本地运行简单，适合课程作业提交和截图展示。
