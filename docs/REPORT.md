# LwChat 课程报告初稿

## 1. 项目背景

即时聊天是 Web 应用中常见的功能，通常涉及用户认证、消息保存、实时通信、文件上传和前端交互设计。为了理解这些技术在实际项目中的组合方式，本项目实现了一个轻量级聊天系统 LwChat。

LwChat 面向课程作业场景，重点是本地可运行、功能可演示、代码结构清晰。系统没有追求商业级即时通信产品的完整复杂度，而是围绕私聊、群聊、图片消息和历史消息加载实现核心闭环。

## 2. 需求分析

系统需要满足以下基本需求：

- 用户可以注册、登录和退出。
- 用户可以查看其他用户，并进入一对一私聊。
- 私聊支持文本消息和图片消息。
- 消息需要通过 WebSocket 实时收发。
- 消息需要保存到数据库，刷新或重新登录后可以查看历史记录。
- 用户可以创建群聊、加入群聊并发送群消息。
- 群聊需要展示成员列表、群主身份和群公告。
- 群主可以修改群公告，并移除普通成员。
- 前端界面需要接近轻量版微信/QQ 聊天软件，适合演示和截图。

## 3. 系统设计

系统采用前后端分离结构。

前端使用 Vue3 和 Vite，主要负责页面渲染、会话切换、消息展示、图片选择和 WebSocket 连接。页面采用三栏布局：左侧为会话栏，中间为聊天区，右侧为信息栏。

后端使用 Django、Django REST Framework 和 Django Channels。普通业务通过 HTTP API 完成，例如注册、登录、会话列表、历史消息、群管理和图片上传。实时消息通过 WebSocket 完成，后端 Consumer 收到消息后保存到数据库，再推送给对应用户或群聊房间。

数据存储使用 SQLite，适合本地演示和课程提交。图片文件保存到后端本地 `media/chat_images/` 目录。

## 4. 数据库设计

系统主要使用以下数据表：

- `User`：Django 内置用户表，用于保存用户名、密码和昵称。
- `PrivateMessage`：保存私聊消息，包括发送者、接收者、内容、消息类型、图片地址、创建时间。
- `ChatGroup`：保存群聊信息，包括群名称、群主、群公告。
- `GroupMember`：保存群成员关系，包括用户、群聊和角色。
- `GroupMessage`：保存群聊消息，包括群聊、发送者、内容、消息类型、图片地址和创建时间。

其中消息类型使用 `text` 和 `image` 区分文本消息与图片消息。图片消息同时保存图片 URL，前端根据类型决定渲染文本气泡还是图片气泡。

## 5. 核心功能实现

### 用户注册和登录

用户注册和登录通过 HTTP 接口完成。注册时后端创建 Django 用户，登录时使用 Django 的认证能力校验用户名和密码。前端登录成功后，将用户基本信息保存到 `localStorage`，刷新页面后可以继续保持当前演示用户状态。

### 私聊功能

私聊流程如下：

1. 用户点击左侧私聊会话。
2. 前端通过 HTTP 获取历史消息。
3. 前端建立当前用户的私聊 WebSocket 连接。
4. 发送消息时，前端通过 WebSocket 发送消息数据。
5. 后端保存消息并推送给发送者和接收者。
6. 前端收到消息后追加到消息列表，并自动滚动到底部。

### 群聊功能

用户可以创建群聊，创建者自动成为群主。其他用户点击群聊后可以加入。群聊消息通过群聊 WebSocket 房间广播。群成员列表和群公告通过 HTTP 接口加载。

群管理采用简化实现，群主可以修改公告和移除普通成员。

## 6. WebSocket 实时通信说明

系统使用 Django Channels 实现 WebSocket。

私聊连接地址：

```txt
ws://localhost:8000/ws/private/<当前用户ID>/
```

群聊连接地址：

```txt
ws://localhost:8000/ws/group/<群聊ID>/
```

私聊中，每个用户加入自己的用户房间。后端收到消息后保存到 `PrivateMessage`，再分别推送到发送者和接收者房间。群聊中，所有进入同一群聊的连接加入同一个群房间，后端保存 `GroupMessage` 后广播给房间内连接。

当前项目使用 `InMemoryChannelLayer`，适合本地单进程演示，不需要 Redis。

## 7. 图片消息实现说明

图片消息采用 HTTP 上传和 WebSocket 发送结合的方式。

1. 前端用户选择图片。
2. 前端通过 `POST /api/upload-image/` 上传图片文件。
3. 后端将图片保存到 `backend/media/chat_images/`。
4. 后端返回图片 URL。
5. 前端通过 WebSocket 发送一条 `message_type=image` 的消息。
6. 聊天窗口根据消息类型展示图片。

这种方式比直接通过 WebSocket 传输图片文件更简单，也更适合课程项目演示。

## 8. 离线消息实现说明

系统没有实现复杂的离线推送。当前采用简化实现：所有私聊和群聊消息都会保存到 SQLite 数据库中。

当用户不在线时，其他用户发送的消息仍会正常写入数据库。该用户重新登录或刷新页面后，前端通过历史消息接口加载数据库中的记录，从而体现“离线期间消息不丢失”。

## 9. 系统测试

测试内容包括：

- 后端迁移检查：`python manage.py makemigrations`、`python manage.py migrate`
- 后端配置检查：`python manage.py check`
- 前端构建检查：`npm run build`
- 注册和登录流程测试
- 私聊文本消息实时收发测试
- 私聊图片上传和实时展示测试
- 群聊创建、加入和消息发送测试
- 刷新页面后历史消息加载测试

测试结果表明，项目可以完成课程演示所需的主要流程。

## 10. 项目总结

LwChat 实现了一个轻量级聊天系统的核心功能，包括用户认证、私聊、群聊、图片消息、消息持久化和 WebSocket 实时通信。系统使用 SQLite 和 InMemoryChannelLayer，降低了部署和演示成本。

项目仍有简化之处，例如认证方式使用本地演示方案，未实现商业级好友关系、复杂未读状态和多进程消息层。但对于课程作业而言，系统已经形成完整闭环，能够清晰展示 Web 聊天系统的基本设计和实现过程。
